# Discord Delete Post Bot

自分の投稿だけを指定範囲で削除する Discord bot です。コマンド実行者以外の投稿は、削除候補収集時と削除直前の両方で除外します。

## セットアップ

1. Python 3.10 以上を用意します。
2. 依存関係をインストールします。

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

3. `.env.example` を参考に `.env` を作成します。

```bash
cp .env.example .env
```

4. Discord Developer Portal で bot を作成し、`.env` の `DISCORD_TOKEN` に bot token を設定します。

5. bot をサーバーへ招待します。必要な権限は次の通りです。

Discord Developer Portal の OAuth2 URL Generator で `bot` と `applications.commands` を選び、Bot Permissions では以下だけを ON にしてください。

一般権限:

- `チャンネルを表示` / `View Channels`

テキストの権限:

- `メッセージを送る` / `Send Messages`
- `メッセージを管理` / `Manage Messages`
- `メッセージ履歴を読む` / `Read Message History`
- `スラッシュコマンドを使用` / `Use Application Commands`

`管理者`、`サーバー管理`、`チャンネルの管理`、`ロールの管理` は不要です。

6. 起動します。

```bash
python bot.py
```

## 起動時のエラー

`403 Forbidden (error code: 50001): Missing Access` が出る場合は、スラッシュコマンドをサーバーへ同期できていません。次を確認してください。

- `.env` の `DISCORD_GUILD_ID` が bot を入れたサーバーの ID になっている
- bot を招待するときに OAuth2 URL Generator で `bot` と `applications.commands` の両方を選んでいる
- bot がそのサーバーに参加済み
- 招待URLを作り直して、もう一度サーバーへ追加している

`DISCORD_GUILD_ID` を空にするとグローバルコマンドとして登録しますが、Discord 側の反映に時間がかかる場合があります。

## コマンド

`/delete_my_posts`

指定したチャンネル内で、コマンドを実行した本人の投稿だけを削除します。

主な引数:

- `開始日時`: 削除範囲の開始日時。例: `2026-06-30 09:00`。省略すると取得できる最古側まで対象です。
- `終了日時`: 削除範囲の終了日時。例: `2026-06-30 18:30`。省略すると最新側まで対象です。
- `対象チャンネル`: 削除対象のチャンネル。省略するとコマンドを実行したチャンネルが対象です。
- `全チャンネル`: `はい` にすると、サーバー内で bot が閲覧できる全テキストチャンネルを対象にします。
- `確認件数上限`: 最大で確認するメッセージ数。初期値は `1000`、`0` にすると取得可能な履歴を上限なしで確認します。
- `ピン留め含む`: ピン留め済みの自分の投稿も削除するか。`はい` / `いいえ` から選びます。

日時はタイムゾーンを書かない場合、日本時間として扱います。日付だけを指定した場合は `2026-06-30 00:00:00` として扱いますが、`終了日時` の日付だけ指定はその日の終わりまで含めます。

削除実行は1ユーザーにつき1日3回までです。確認画面を表示しただけでは回数に含めません。確認画面で「実行」を押したときだけ回数に含めます。回数は `delete_usage.json` に保存されます。

## 安全な使い方

コマンドを実行すると、削除候補数を表示する確認画面が出ます。削除する場合は「実行」、中止する場合は「拒否」を押してください。

日時範囲を指定して削除する例:

```text
開始日時 = 2026-06-30 09:00
終了日時 = 2026-06-30 18:30
対象チャンネル = #general
```

チャンネル内の取得可能な履歴をすべて対象にしたい場合は、`開始日時` と `終了日時` を空にし、`確認件数上限` を `0` にしてください。まず確認画面の候補数を必ず確認してください。

全テキストチャンネルを対象にして日時範囲で削除する場合:

```text
全チャンネル = はい
開始日時 = 2026-06-30 09:00
終了日時 = 2026-06-30 18:30
```

全テキストチャンネルを対象にして取得可能な自分の投稿を全削除する場合:

```text
全チャンネル = はい
開始日時 =
終了日時 =
確認件数上限 = 0
```

`全チャンネル = はい` と `対象チャンネル` は同時に指定できません。全チャンネル対象の場合、`確認件数上限` は各チャンネルごとの確認上限として扱われます。

## 注意

- Discord API の制約上、bot から見て他ユーザーの投稿削除になるため `Manage Messages` 権限が必要です。
- `全チャンネル = はい` の場合も、削除されるのはコマンド実行者本人の投稿だけです。
- `確認件数上限` を大きくすると処理に時間がかかります。
- 投稿削除は取り消せませんので自己責任で行ってください。
